Testing and content fixture
===========================

How to create a defined content for testing? Ideas:

* Write sql-statements in files
* Use Faker https://faker.readthedocs.io/en/master/

     Faker is a Python package that generates fake data for you. Whether you 
     need to bootstrap your database, create good-lookin  g XML documents, 
     fill-in your persistence to stress test it, or anonymize data taken 
     from a production service, Faker is for you.

* https://github.com/TildBJ/seeder

     Seeder is a TYPO3 Extension that generates fake data for your TYPO3 Extension. 
     Its intended for developers only!!! This Version is an experimental version!
